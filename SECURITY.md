# Security Policy

## Reporting Security Vulnerabilities

**Do NOT open public issues for security vulnerabilities.** Instead, please use GitHub's Private Vulnerability Reporting feature to disclose security issues responsibly.

### Report a Vulnerability

1. **Preferred Method:** Use GitHub's built-in private advisory feature.
   - Go to the Security tab of the repository and choose "Report a vulnerability".
   - Describe the vulnerability in detail, including steps to reproduce.

2. **Alternative:** Email the maintainers directly
   - Email: rutova2@gmail.com
   - Include a detailed description of the vulnerability and reproduction steps.

### Response Timeline

- **Acknowledgment**: We aim to acknowledge receipt of your report within 48 hours.
- We will conduct a thorough assessment and work on a fix as quickly as possible.
