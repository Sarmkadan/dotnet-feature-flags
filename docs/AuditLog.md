# AuditLog

The `AuditLog` type records changes made to feature flags within the dotnet-feature-flags system. It captures metadata about modifications, including the actor, timestamp, and details of the change, enabling traceability and auditability of feature flag state transitions.

## API

### `public int Id`
A unique identifier for the audit log entry. Assigned by the persistence layer and immutable after creation.

### `public int FeatureFlagId`
The identifier of the feature flag that was modified. Corresponds to the `Id` of a `FeatureFlag` entity.

### `public AuditAction Action`
The type of action performed on the feature flag. Values are defined by the `AuditAction` enum (e.g., `Create`, `Update`, `Delete`, `Rollback`).

### `public string ChangedBy`
The identifier of the user or system responsible for the change (e.g., username, service principal ID). Not validated by the type itself.

### `public DateTime ChangedAt`
The UTC timestamp when the change occurred. Set by the system at the time of the action.

### `public string OldValue`
The previous state of the feature flag, serialized as a string. Format depends on the flag type (e.g., boolean `"true"`/`"false"`, string `"v1.0"`, or numeric `"42"`). May be `null` for creation actions.

### `public string NewValue`
The new state of the feature flag, serialized as a string. Format matches `OldValue`. May be `null` for deletion actions.

### `public string Description`
A human-readable explanation of the change. Provided by the caller or generated by the system (e.g., "Enabled feature for beta users").

### `public string? IpAddress`
The IP address of the client that initiated the change, if available. May be `null` for server-side or system-initiated actions.

### `public string? UserAgent`
The user agent string of the client that initiated the change, if available (e.g., browser or application identifier). May be `null` for automated processes.

### `public FeatureFlag? FeatureFlag`
A reference to the full `FeatureFlag` entity associated with this log entry, if loaded. May be `null` if not eagerly fetched.

### `public string GetSummary()`
Returns a concise, human-readable summary of the audit log entry.
**Returns:** A string combining `ChangedBy`, `Action`, `FeatureFlagId`, and `ChangedAt` in a standardized format.
**Throws:** No exceptions.

### `public bool IsRollbackOf()`
Determines whether this log entry represents a rollback of a previous change.
**Returns:** `true` if `Action` is `Rollback` and a prior log entry with matching `FeatureFlagId` and `NewValue` exists; otherwise, `false`.
**Throws:** No exceptions.

### `public bool IsValid()`
Validates the internal consistency of the audit log entry.
**Returns:** `true` if required fields (`FeatureFlagId`, `Action`, `ChangedBy`, `ChangedAt`) are non-null/empty and `ChangedAt` is not in the future; otherwise, `false`.
**Throws:** No exceptions.

### `public (string oldState, string newState) GetChangeDetails()`
Extracts the before-and-after states of the feature flag as a tuple.
**Returns:** A tuple where `oldState` is the value of `OldValue` (or `"null"` if `null`) and `newState` is the value of `NewValue` (or `"null"` if `null`).
**Throws:** No exceptions.
