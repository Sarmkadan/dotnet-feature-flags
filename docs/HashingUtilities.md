# HashingUtilities

The `HashingUtilities` class provides a centralized set of static methods for performing cryptographic hashing, password security operations, and deterministic bucketing calculations within the `dotnet-feature-flags` ecosystem. It encapsulates standard .NET cryptographic algorithms such as SHA-256, SHA-512, MD5, FNV-1a, and HMAC, offering a simplified interface for generating hashes required for feature flag targeting, user segmentation, and secure credential verification without exposing low-level cryptographic implementation details.

## API

### `ComputeSha256`
Computes the SHA-256 hash of the provided input data.
*   **Parameters**: Takes a `string` or `byte[]` input (depending on specific overload implementation context) representing the data to be hashed.
*   **Return Value**: Returns a `string` containing the hexadecimal representation of the 256-bit hash.
*   **Exceptions**: Throws `ArgumentNullException` if the input is null.

### `ComputeSha512`
Computes the SHA-512 hash of the provided input data.
*   **Parameters**: Takes a `string` or `byte[]` input representing the data to be hashed.
*   **Return Value**: Returns a `string` containing the hexadecimal representation of the 512-bit hash.
*   **Exceptions**: Throws `ArgumentNullException` if the input is null.

### `ComputeHashBucket`
Calculates a deterministic integer bucket index based on the input data, typically used for percentage-based feature rollouts.
*   **Parameters**: Accepts the data to be hashed (usually a user ID or context string) and the total number of buckets (range).
*   **Return Value**: Returns an `int` representing the bucket index, guaranteed to be within the range `[0, totalBuckets)`.
*   **Exceptions**: Throws `ArgumentOutOfRangeException` if the total number of buckets is less than or equal to zero.

### `ComputeMd5`
Computes the MD5 hash of the provided input data.
*   **Parameters**: Takes a `string` or `byte[]` input representing the data to be hashed.
*   **Return Value**: Returns a `string` containing the hexadecimal representation of the 128-bit hash.
*   **Exceptions**: Throws `ArgumentNullException` if the input is null. *Note: MD5 is provided for legacy compatibility and should not be used for new security-sensitive applications.*

### `ComputeFnv1aHash`
Computes the FNV-1a (Fowler-Noll-Vo) non-cryptographic hash of the input data.
*   **Parameters**: Takes a `string` or `byte[]` input.
*   **Return Value**: Returns a `uint` representing the 32-bit FNV-1a hash value.
*   **Exceptions**: Throws `ArgumentNullException` if the input is null. This method is optimized for speed and distribution in hashing scenarios where cryptographic strength is not required.

### `HashPassword`
Generates a secure hash for a password, including salt generation and iteration logic.
*   **Parameters**: Takes a `string` representing the plain-text password.
*   **Return Value**: Returns a `string` containing the formatted hash result, which includes the salt and iteration count embedded within the output string.
*   **Exceptions**: Throws `ArgumentNullException` if the password is null or empty.

### `VerifyPassword`
Verifies a plain-text password against a previously generated hash.
*   **Parameters**: Takes two strings: the plain-text `password` and the `hashedPassword` (generated by `HashPassword`).
*   **Return Value**: Returns a `bool` indicating `true` if the password matches the hash, or `false` otherwise.
*   **Exceptions**: Throws `ArgumentNullException` if either argument is null. Returns `false` if the hash format is invalid rather than throwing an exception.

### `GenerateSecureHash`
Generates a cryptographically strong random hash, typically used for tokens or secrets.
*   **Parameters**: Optionally accepts an integer specifying the desired length in bytes.
*   **Return Value**: Returns a `string` containing the hexadecimal representation of the generated secure random bytes.
*   **Exceptions**: Throws `ArgumentOutOfRangeException` if the requested length is negative.

### `ComputeHmacSha256`
Computes a Hash-based Message Authentication Code (HMAC) using the SHA-256 algorithm.
*   **Parameters**: Takes the data to be hashed (`string` or `byte[]`) and a `byte[]` or `string` representing the secret key.
*   **Return Value**: Returns a `string` containing the hexadecimal representation of the HMAC.
*   **Exceptions**: Throws `ArgumentNullException` if the data or key is null. Throws `ArgumentException` if the key length is invalid for the algorithm.

## Usage

### Example 1: Deterministic Feature Flag Bucketing
This example demonstrates how to use `ComputeHashBucket` to determine if a specific user falls within a 10% rollout cohort for a new feature.

```csharp
using Microsoft.FeatureFilters.Utilities;

public class FeatureRolloutService
{
    public bool IsUserInRollout(string userId, int totalBuckets = 100)
    {
        if (string.IsNullOrEmpty(userId))
        {
            return false;
        }

        // Compute the bucket for the user ID
        int userBucket = HashingUtilities.ComputeHashBucket(userId, totalBuckets);

        // Assign feature to first 10 buckets (10%)
        return userBucket < 10;
    }
}
```

### Example 2: Secure Configuration Token Verification
This example illustrates generating a secure token and verifying a password hash for administrative access to feature flag configurations.

```csharp
using Microsoft.FeatureFilters.Utilities;
using System;

public class AdminAuthService
{
    public string CreateSessionToken()
    {
        // Generate a 32-byte secure random hash for a session token
        return HashingUtilities.GenerateSecureHash(32);
    }

    public bool AuthenticateAdmin(string inputPassword, string storedHash)
    {
        // Verify the input password against the stored hash
        // The storedHash should have been created via HashPassword previously
        return HashingUtilities.VerifyPassword(inputPassword, storedHash);
    }
    
    public string HashNewAdminPassword(string plainPassword)
    {
        return HashingUtilities.HashPassword(plainPassword);
    }
}
```

## Notes

*   **Thread Safety**: All methods within `HashingUtilities` are static and stateless. They rely on thread-safe underlying .NET cryptographic implementations, making the class fully thread-safe for concurrent use across multiple requests without external locking.
*   **Input Validation**: All methods accepting reference types (`string`, `byte[]`) strictly validate for `null` inputs and will throw `ArgumentNullException`. Callers should ensure inputs are sanitized before invocation.
*   **Algorithm Selection**:
    *   Use `ComputeSha256` or `ComputeHmacSha256` for general-purpose cryptographic hashing and integrity checks.
    *   Use `ComputeFnv1aHash` only for non-security scenarios such as load balancing or internal dictionary keys where performance is critical and collision resistance requirements are lower.
    *   `ComputeMd5` is retained strictly for backward compatibility with legacy systems; it must not be used for new security implementations due to known collision vulnerabilities.
*   **Password Storage**: The `HashPassword` and `VerifyPassword` methods automatically handle salt generation and storage formatting. Do not attempt to manually salt passwords before passing them to these methods, as this will result in verification failures.
*   **Determinism**: `ComputeHashBucket`, `ComputeFnv1aHash`, and the standard hash functions (`Sha256`, `Sha512`, `Md5`) are deterministic. Identical inputs will always produce identical outputs within the same version of the library, which is essential for consistent feature flag targeting.
